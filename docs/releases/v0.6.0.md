# v0.6.0

## Summary

Architecture catches up to the ambition. Three surfaces are
structurally equal, the hex layers are real, symbol identity has a
canonical footing, and the bad-code backlog is empty for the first
time in the project's history.

## What Changed

- **Bad-code burndown**: all 36 structural debt cards retired, plus
  7 code review findings fixed. Zero bad-code items remain.
- **Anti-sludge policy**: semgrep rules, shell checks, and the full
  ANTI_SLUDGE_TYPESCRIPT_POLICY adopted as the quality bar.
- **Explicit execution context**: `ToolHandler` receives `ctx` as a
  parameter instead of relying on `AsyncLocalStorage` magic.
- **Session disambiguation**: "session" no longer means 5 things.
  `GovernorTracker` for budget/tripwires, `RegisteredTransport` for
  daemon connections, `WorkspaceMode` for binding posture.
- **MCP composition decomposition**: server, router, control-plane,
  worker-pool, monitor-runtime, and repo-state all split into focused
  sub-modules. No file over 450 lines.
- **Security hardening**: path resolver confines absolute paths and
  resolves symlinks. Secret scrubbing across all observability paths.
  `graft_map` respects output caps and session budget.
- **WARP Level 1 deepening**: canonical symbol identity projection
  across diff/since surfaces. Qualified symbol names prevent
  same-named method collisions. Indexer errors propagate instead of
  producing silently wrong graphs.
- **Attribution hardening**: transport session, environment inference,
  and session continuity strategies make `actor:unknown` the exception.
- **Tool-call provenance footprints**: paths, symbols, and line
  regions recorded per tool invocation.
- **Dynamic project root**: `GRAFT_PROJECT_ROOT` env var lets MCP
  servers inspect repos other than their own.
- **Implicit daemon binding**: repo-scoped tools auto-bind when path
  evidence exists, removing the mandatory `workspace_bind` step.

## Why It Matters

v0.5.0 was "the activity layer ships." v0.6.0 is "the architecture
catches up to the ambition." Three surfaces are structurally equal,
the hex layers are real and mechanically enforced, symbol identity has
a canonical footing, and local history attribution tells you who
actually drove a slice of work.

For agents: explicit context threading, provenance footprints, and
governed `graft_map` output make the tool surface more predictable and
less likely to blow up context windows.

For operators: the anti-sludge policy and code review fixes mean the
codebase is cleaner, more honest, and harder to regress.

## Breaking Changes

- `ToolHandler` signature changed from `(args)` to `(args, ctx)`.
  `createHandler(ctx)` changed to `createHandler()`. Any custom tool
  handlers must update.
- `SessionTracker` renamed to `GovernorTracker`. `ctx.session` renamed
  to `ctx.governor`. Internal-only â€” wire format (`sessionId` etc.)
  unchanged.
- `graft_map` now truncates at 100 files or 50KB and returns
  `truncated: true` with guidance. Previously returned unbounded.

## Migration

- Update any custom tool handlers to the new `(args, ctx)` signature.
- Replace `ctx.session` references with `ctx.governor`.
- If you depend on `graft_map` returning full unbounded results,
  narrow your `path` query or use `summary: true` mode.
- Set `GRAFT_PROJECT_ROOT` in your MCP server config if graft should
  inspect a repo other than the one it's installed in.
