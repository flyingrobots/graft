# v0.7.1

## Summary

v0.7.1 is a patch release for npm package hygiene. It keeps the
published package focused on built runtime artifacts, removes the
published TypeScript source tree from the package file list, and moves
`tsx` back to development-only tooling.

## What Changed

- The npm package no longer publishes `src/`.
- `bin/graft.js` now starts the built CLI from
  `dist/cli/entrypoint.js` instead of re-executing through `tsx`.
- Generated Claude hook commands now target built hook entry points in
  `dist/hooks/*.js`.
- `graft init --write-claude-hooks` migrates generated v0.7.0 Claude
  hook commands from `src/hooks/*.ts` to `dist/hooks/*.js`.
- Daemon child workers use compiled JavaScript workers when running
  from `dist`, keeping `tsx` out of published runtime dependencies.
- `tsx` moved from `dependencies` to `devDependencies`.
- `package.json` now declares `"sideEffects": false` for bundlers.
- Manual publish attempts now run `pnpm release:check` through
  `prepublishOnly`.

## Why It Matters

The published package now has a cleaner runtime contract: consumers get
the built CLI, API types, and documented setup surfaces without an
invitation to rely on internal TypeScript source paths.

## Migration

No setup migration is required for standard CLI, MCP, or API users.

If you copied Claude hook commands manually from older setup docs,
rerun:

```bash
npx @flyingrobots/graft init --write-claude-hooks
```

That command upgrades generated v0.7.0 hook commands from
`node --import tsx node_modules/@flyingrobots/graft/src/hooks/*.ts` to
`node_modules/@flyingrobots/graft/dist/hooks/*.js` and preserves
unrelated hook commands.
