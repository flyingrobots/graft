# v0.9.0

## Summary

v0.9.0 is a Schema Authority release. It establishes Graft's side of the
future Echo integration contract: a canonical GraphQL structural-history
schema compiled by a pinned Wesley CLI, a deterministic Echo package
descriptor, explicit evidence labels that keep current git-warp readings
from masquerading as Echo-native witnesshood, and a fully tested fake
Echo-shaped TypeScript witness proving the transport seam, codecs, typed
client, and obstruction-to-posture mapping end to end.

This release makes no Echo runtime claim and no durability claim. The
Echo-facing stack is internal, exercised only by tests, and barred from
production contexts by guard tests. Public command and API surfaces are
unchanged except for additive MCP workspace tools.

## What Changed

- `schemas/graft-structural-history.graphql` is the canonical
  structural-history contract, including the schema's first Intent
  (`recordGitWarpImportBatch`) and source-span invariants. The hermetic
  gate (`pnpm schema:structural-history:check`) regenerates the
  Wesley-generated TypeScript model and LE-binary var codecs with the
  pinned Wesley CLI 0.0.5 and byte-compares them against the committed
  artifacts.
- `schemas/graft-structural-history.echo-package.json` describes the
  package Graft expects Echo to host later, with deterministic schema,
  artifact, and generator identities and a drift checker.
- `graft_review` impact counts and `graft_dead_symbols` read through the
  Graft-owned `StructuralReadingPort` carrying `echo-native`,
  `git-warp-imported`, or `fallback-translated` evidence labels;
  response shapes are unchanged.
- New internal Echo witness stack: `EchoKernelTransport` byte seam,
  EINT v1 + canonical-CBOR envelope codecs, typed structural-history
  client with Echo's intent outcome kinds, an Echo-backed reading
  adapter with the contract obstruction → posture mapping, and a
  deterministic fake transport that enforces wire-level authority
  (`FORBIDDEN_CONTROL_INTENT`).
- MCP sessions can open and activate additional git worktrees through
  `workspace_open` and inspect them with `workspace_list_opened`.
- `graft_blame` and `graft symbol history` carry per-version path and
  line-range facts from WARP symbol timelines.
- Reference-count readings report a partial residual posture when
  committed import-scan fallback evidence is unavailable.

## Known Limits

- No real Echo runtime is contacted; the witness proves the Graft side
  only. The integration gate (after the generated-model parity slice)
  decides the real transport.
- Parser-dependent search tools can fail on the first call of a fresh
  session (`ensureParserReady` ordering defect, tracked in the backlog).
