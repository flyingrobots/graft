# v0.5.0

## Summary

Precision tools, a complete grouped CLI peer surface, Markdown-aware
bounded reads, and a much tighter release/security posture.

## What Changed

- **Precision tools**: `code_show` and `code_find` let agents and
  operators jump directly to symbols instead of grepping and opening
  whole files.
- **Layered worldline semantics**: MCP responses now distinguish commit
  history, ref views, and live workspace state, so tools can say more
  honestly where an answer came from.
- **Markdown summaries**: bounded-read tools now understand Markdown
  headings and section ranges instead of treating docs as unsupported
  text.
- **CLI parity surface**: the CLI now mirrors the core MCP surface
  under grouped namespaces:
  - `read`
  - `struct`
  - `symbol`
  - `diag`
- **One-step bootstrap**: `graft init` can now write project-local
  Claude and Codex config directly with explicit flags.
- **Explicit MCP startup**: `graft serve` is now the durable stdio
  startup path, while bare CLI invocation defaults to help for humans.
- **Versioned machine-readable outputs**: MCP and CLI peer commands now
  include versioned `_schema` metadata.
- **Release hardening**: `run_capture` has an explicit trust posture,
  release preflight includes a dependency/security gate, and the
  previous Vite advisory path is fixed.

## Why It Matters

This release makes Graft feel much more like a complete product surface
and much less like a promising substrate you need to mentally assemble
yourself.

Agents get better precision and more honest structure. Operators get a
more coherent CLI/MCP setup story. Contributors get stronger contracts
and a clearer architecture map.

## Breaking Changes

None.

## Migration

- Prefer `npx @flyingrobots/graft serve` anywhere you intend to start
  the MCP server explicitly.
- Use `graft init --write-claude-mcp --write-claude-hooks` or
  `graft init --write-codex-mcp` for project-local bootstrap instead of
  manual config edits.
- For stricter environments, disable shell capture with
  `GRAFT_ENABLE_RUN_CAPTURE=0` and disable persisted capture logs with
  `GRAFT_RUN_CAPTURE_PERSIST=0`.
