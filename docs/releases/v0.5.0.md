# v0.5.0

## Summary

Bounded between-commit activity for humans and agents, backed by
persisted local artifact history, explicit attribution, reactive
workspace footing, semantic transition meaning, and same-repo
concurrency posture.

## What Changed

- **Precision tools**: `code_show` and `code_find` let agents and
  operators jump directly to symbols instead of grepping and opening
  whole files.
- **Between-commit activity view**: bounded `activity_view` over local
  `artifact_history`, plus the human-facing CLI peer
  `graft diag activity`.
- **Persisted local history**: continuity, stage, transition, and read
  activity survive reconnects as bounded local history.
- **Attribution**: bounded confidence and evidence now travel with
  local activity instead of being left implicit.
- **Reactive workspace footing**: checkout-boundary hook posture,
  forked-vs-stable lineage, and degraded local-edit coverage are now
  explicit product truth.
- **Semantic transitions**: merge/rebase/conflict/bulk/index posture is
  summarized as bounded semantic meaning, not just raw Git lifecycle.
- **Same-repo concurrency**: shared repo/worktree and divergent
  checkout posture are visible without pretending multi-writer
  provenance is solved.

## Why It Matters

This release makes Graft feel like a real between-commit memory product
instead of only a bounded-read substrate.

Humans can now inspect recent work without reconstructing chat logs.
Agents keep the bounded machine-readable surface they need. Operators
get a clearer footing/transition/concurrency story. Contributors get a
repo that is paused cleanly for release execution instead of carrying
another active cycle.

## Breaking Changes

None expected for normal users. This is a minor release with broader
surface area, not a contract-breaking one.

## Migration

- Prefer `graft diag activity` or MCP `activity_view` when you want a
  bounded answer to “what happened between the last Git commit and
  now?”
- Continue using `graft serve` for stdio MCP startup and `graft init
  --write-*` for project-local bootstrap.
- No migration is required for existing read / struct / symbol users.
