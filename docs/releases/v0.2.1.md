# v0.2.1

## Summary

Hooks and hardening. Claude Code agents are now governed at the hook
level, and all domain types are frozen value objects with constructor
validation.

## What Changed

- **Claude Code hooks**: PreToolUse hook blocks banned files (secrets,
  binaries, lockfiles, `.graftignore` matches) before Read executes.
  PostToolUse hook shows agents the context cost of large file reads
  and suggests `safe_read` as an alternative.
- **Value objects**: OutlineEntry, JumpEntry, DiffEntry, OutlineDiff,
  and Tripwire are now frozen classes with constructor validation and
  forgery protection. Completes the runtime-truth migration for all
  domain types.
- **Bug fixes**: trim-and-validate for value object names, UTF-8
  buffer safety in hook stdin reader, path traversal guard in hooks,
  Node engine minimum bumped to `>=20.11.0`.

## Why It Matters

Hooks make graft's policy enforcement automatic for Claude Code users
once hooks are configured in `.claude/settings.json` — no per-agent
opt-in required after setup. Value objects eliminate a class of bugs
where malformed data could pass through plain-object literals.

## Breaking Changes

None.

## Migration

No migration required. Configure hooks once via `.claude/settings.json`
(already committed to the repo) and they apply automatically to all
Claude Code sessions.
