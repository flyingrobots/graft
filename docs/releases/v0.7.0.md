# v0.7.0

## Summary

v0.7.0 turns Graft's structural-history layer from a commit-walking
prototype into WARP-backed product behavior, then stabilizes the agent
runtime around that truth. The release branch now indexes bounded HEAD
snapshots into the WARP graph, records cross-file import/reference
edges, exposes daemon-backed MCP runtime selection, ships a read-only
daemon status surface, and adds the first governed exact-edit MCP tool.

## What Changed

### WARP Structural History

- **WARP snapshot indexing**: `indexHead` is the active indexing path.
  It emits commit, file, directory, symbol, and compact AST anchor facts
  with tick metadata. Full AST snapshots are attached as blobs instead
  of being materialized as graph bulk.
- **Cross-file reference graph**: import specifiers, re-exports, and
  namespace imports now produce graph edges that can answer reference
  questions without ripgrep.
- **Structural operations on WARP**: `graft_log`, `graft_churn`,
  `graft_blame`, and `graft_review` use WARP graph data in their MCP
  execution paths. `graft_churn` now uses aggregate-backed WARP queries
  for live-symbol change counts and tick receipts for removed-symbol
  churn.
- **Provenance-backed structural blame**: `graft_blame` now hydrates
  structural blame from WARP provenance ticks rather than the old
  timeline-derived last-touch path.
- **Slice-first progress**: high-risk `getEdges()` / `getNodes()` paths
  in `references.ts`, structural queries, and precision WARP tooling
  have moved to `traverse`, `QueryBuilder`, or tick receipts.
- **Lazy index guardrails**: unbounded whole-repo indexing is refused,
  per-file patch payloads are capped, and `graft index --path <path>`
  is the explicit refresh path.
- **Sludge detection**: `doctor` has an opt-in parser-backed structural
  smell scan for typedef/class imbalance, JSDoc cast density, homeless
  constructor functions, free functions operating on project types, and
  high-symbol-count files.
- **Refactor difficulty scoring**: agents can call `graft_difficulty`
  or `graft symbol difficulty` for a per-symbol scalar built from WARP
  churn curvature and reference-edge friction.
- **Docs integrity tools**: stale-doc checking, drift sentinel checks,
  structural drift detection, symbol timelines, and dead-symbol
  detection now exist as WARP-aware support tools.
- **Export surface semver impact**: `graft_exports` now classifies
  breaking exported signature changes as `major` instead of treating
  every signature change as `patch`.

### Agent Runtime Surfaces

- **Daemon-backed stdio MCP**: `graft serve --runtime daemon` exposes
  the tested daemon bridge as a release-facing runtime. Generated MCP
  config can select repo-local stdio or daemon-backed stdio through
  `graft init --mcp-runtime daemon --write-*-mcp`.
- **Read-only daemon status**: `graft daemon status [--socket <path>]`
  renders daemon health, session counts, workspace posture, monitor
  summary, scheduler pressure, and worker pressure without adding
  mutating daemon actions.
- **Git-facing structural review**: `git graft enhance --since <ref>`
  and `git-graft enhance --since <ref>` compose shipped `graft_since`
  and `graft_exports` facts into a concise human summary or
  schema-validated JSON.
- **Governed exact edit**: `graft_edit` is a narrow MCP exact
  replacement tool. It accepts `path`, `old_string`, and `new_string`,
  refuses outside-repo and policy-denied paths, succeeds only on one
  exact match, uses the filesystem/path ports, and does not create,
  delete, append, rename, chmod, format, or multi-edit.
- **Advisory agent drift warning**: successful `graft_edit` calls can
  emit optional `driftWarnings` when a same-session edit removes then
  later reintroduces the narrow `jsdoc_typedef` structural pattern. The
  warning is advisory only and does not claim causal write provenance.

### Release Hardening

- **Dockerized test isolation**: the default `pnpm test` path now runs
  in a copy-in Docker test container, so validation does not inherit the
  operator checkout's Git hooks or Git environment.
- **Repo path confinement hardening**: repo path resolution now checks
  the nearest existing ancestor, preventing future write/create tools
  from escaping through an existing symlinked parent directory.
- **Backlog DAG truth**: the checked-in backlog dependency DAG now
  renders `blocked_by`, `blocking`, and `blocked_by_external` metadata
  from card frontmatter.
- **Post-v0.7.0 scope cleanup**: full LSP semantic enrichment and the
  remaining medium-risk slice-first read sweep are not active v0.7.0
  blockers. They are preserved as post-v0.7.0 follow-up cards.
- **Agent worktree hygiene**: release preflight and the repo-local
  pre-commit hook now reject tracked or staged `.claude/worktrees/`
  paths before agent scratch worktrees can become commit history.

## Why It Matters

Graft can now answer structural-history questions from graph facts
rather than repeatedly walking Git history or grepping source text. It
also gives operators a clearer daemon runtime choice and gives agents a
first governed edit aperture without pretending full governed writes or
causal write provenance are complete.

For operators, v0.7.0 also makes release truth more explicit: the
branch, package version, changelog, release notes, BEARING, and METHOD
release packet now agree about the current version line and about which
large WARP follow-ups are intentionally post-release.

## Breaking Changes

- `graft_blame` has a new WARP-backed result shape. It returns
  tick-aware symbol timeline data and a simplified `createdInCommit`
  value. Consumers that parse the old MCP result shape must update.
- The legacy commit-walking WARP indexer was removed. Internal callers
  should use `indexHead`.

Graft is still pre-1.0, so command and MCP schema changes can ship in
minor releases, but externally meaningful shape changes are called out
explicitly.

## Migration

- If you only use Graft through the documented CLI/MCP tools, no setup
  migration is required.
- If you parse `graft_blame` responses directly, update to the new
  timeline-oriented WARP result.
- If you called internal indexing helpers, migrate from `indexCommits`
  to `indexHead`.
- If you use `graft_exports` for release automation, expect a more
  conservative `semverImpact` when exported function signatures remove
  parameters, add required parameters, change parameter types, or
  change return types.
- If you want daemon-backed MCP, choose it explicitly with
  `graft serve --runtime daemon` or generated config from
  `graft init --mcp-runtime daemon --write-*-mcp`. Plain `graft serve`
  remains repo-local stdio.
- If you consume `graft_edit`, treat `driftWarnings` as advisory
  diagnostics only. They are not policy refusals and are not causal
  write-provenance events.
