# Cycle 0043 — Code Of Conduct

**Type:** Docs  
**Legend:** CORE

## Hill

When someone lands on the repo as a contributor or reviewer, the
project has a clear, conventional participation standard instead of
silence about conduct expectations.

## Playback Questions

### Contributor

1. Can I find a code of conduct from the normal repo entry points?
2. Does it tell me what behavior is expected and how to report a
   problem?

### Maintainer

1. Is the code of conduct concise and conventional enough for a public
   Apache-licensed developer tool?
2. Does it use a real reporting path instead of placeholder text?

## Scope

- add `CODE_OF_CONDUCT.md`
- link it from `README.md` and `CONTRIBUTING.md`
- close the consumed backlog item

## Non-goals

- creating a large governance packet
- changing contribution workflow or security policy

## Success Criteria

- `CODE_OF_CONDUCT.md` exists at the repo root
- it contains clear expectations, unacceptable behavior, and a real
  reporting path
- `README.md` and `CONTRIBUTING.md` both link to it
