# Cycle 0019: Live Study Design

## Hill

A study methodology that a skeptic would accept. Pre-registered
hypotheses, paired design, automated data collection, explicit
failure modes. The deliverable is the protocol doc, not the results.

## Sponsor

The BEARING: measurement. Graft is a thesis until we have
prospective data. This cycle designs the experiment.

## Deliverable

`docs/study/protocol.md` — a complete study protocol covering:

- Hypotheses (pre-registered, falsifiable)
- Design (paired within-subject)
- Metrics (5, with operational definitions)
- Data collection (automated where possible)
- Analysis plan (statistical tests chosen before data)
- Threats to validity (and mitigations)
- Task bank (real tasks, acceptance criteria)
- Pilot plan (small N first, expand if warranted)

## Non-goals

- Running the study (that's execution, not design)
- Building measurement infrastructure (separate cycle)
- Analyzing historical Blacklight data (already done)

## Playback questions

**Human**: Could I hand this protocol to an independent researcher
and have them run it without asking me clarifying questions?

**Agent**: Are the metrics operationally defined with enough
precision that I could compute them from receipts and logs without
human judgment?

Effort: M (design cycle — docs, not code)
