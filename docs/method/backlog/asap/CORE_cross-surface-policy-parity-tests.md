# Cross-surface policy parity tests

Add matrix-driven tests that exercise the same denied file across
multiple entry points and assert equivalent handling.

Minimum cases:

- banned binary
- secret file
- `.graftignore` match
- session-depth outline pressure
- budget pressure
- historical/git-backed read denial

Surfaces:

- Claude Read hooks
- MCP bounded-read tools
- precision tools
- structural tools once they are governed

Done when:

- the audit matrix has executable witnesses
- regressions are caught before readiness claims drift again
