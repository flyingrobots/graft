# FIX: Colorful IR numeric outline IDs Retro

## Summary

Fixed Graft's Colorful prose projector so it accepts the actual
`colorful.syntax/v1` outline ID shape. The decoder now reads integer
`nodeId` and `childNodeIds` values, rejects negative or non-integer IDs, and
normalizes valid IDs to Graft's existing internal string-keyed outline maps.

The cycle also corrected Colorful fixtures in projector, CLI-adapter, MCP tool,
and MCP cache tests so the test corpus matches real `colorful ir` output
instead of a string-ID shadow contract.

## Playback Witness

Artifacts under `docs/method/retro/FIX_colorful-ir-numeric-outline-ids/witness`.

## What surprised you?

The published Graft adapter's version gate was correct, but the fixture model
behind it used string outline IDs and therefore never exercised the public
Colorful CLI's JSON shape.

## What would you do differently?

Future cross-repo adapters should include at least one fixture generated by the
real upstream CLI or an equivalent checked-in golden before release.
